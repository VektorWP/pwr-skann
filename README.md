# PWR Skann — быстрый поиск по power.no со сканером штрих-кодов

## Запуск на GitHub Pages (5 минут)

1. Создай новый **публичный** репозиторий на GitHub (например `pwr-skann`).
2. Залей туда все 6 файлов: `index.html`, `manifest.json`, `sw.js`, `icon-192.png`, `icon-512.png`, `README.md`
   (можно прямо через веб-интерфейс: Add file → Upload files).
3. Settings → Pages → Source: **Deploy from a branch** → Branch: `main`, папка `/ (root)` → Save.
4. Через минуту страница доступна на `https://<твой-ник>.github.io/pwr-skann/`.
5. Открой её в Chrome на Pixel → меню ⋮ → **«Добавить на главный экран» / «Установить приложение»**.

## Первая настройка

1. Нажми ⚙ и проверь **URL-шаблон поиска**. По умолчанию:
   `https://www.power.no/search/?q={q}`
   Сделай один поиск на power.no руками и сверь адресную строку — если путь другой, поправь шаблон
   (`{q}` подставляется автоматически).
2. **Пресеты с фильтрами:** на power.no накликай нужные фильтры (наличие в магазине, цена, бренд…),
   скопируй URL из адресной строки и вставь в новый пресет через ⚙.
   Если в URL фильтров есть поисковый запрос — замени его на `{q}`, тогда пресет будет
   применять фильтры к последнему скану.

## Как пользоваться

- **▶ Start skanner** → навести на штрих-код → вибрация → кнопка «Åpne i Power.no».
- 🔦 — фонарик (если камера поддерживает torch).
- Оранжевый значок ⌕ на пресете = пресет использует `{q}` (последний скан или текст из поля).
- Историю последних 12 сканов можно тапнуть — повторный поиск.
- Тумблер «Åpne automatisk» в ⚙ — открывать power.no сразу после скана без лишнего тапа.

## Отладка на компе

Камера работает только по HTTPS **или** на localhost:
```
cd pwr-skann
python -m http.server 8000
# открой http://localhost:8000
```

## Технические детали

- Сканер: нативный `BarcodeDetector` (Chrome/Android — быстрый, без библиотек).
  Fallback: ZXing с jsDelivr (Firefox и пр.).
- Форматы: EAN-13, EAN-8, UPC-A/E, Code 128, ITF, QR.
- Дедупликация: один и тот же код игнорируется 3 секунды.
- Все настройки, пресеты и история — в localStorage, никуда не отправляются.
- sw.js кэширует оболочку — страница открывается мгновенно и без сети (сам поиск, понятно, требует интернет).

## Версия 2: встроенная выдача + фильтр-пресеты

Теперь результаты рендерятся прямо на странице через внутренний API power.no
(подтверждённый эндпоинт: `https://www.power.no/api/v2/products` — параметр `ids=` точно работает,
`q=` для поиска — предположение, требует проверки).

**Проверка эндпоинта (2 минуты):**
1. Открой power.no на компе → F12 → Network → фильтр Fetch/XHR.
2. Сделай поиск любого товара.
3. Найди запрос, возвращающий JSON со списком товаров → Copy URL.
4. В ⚙ страницы вставь его в «API URL-mal», заменив поисковый текст на `{q}`.

**Фильтр-пресеты** — это клиентские фильтры выдачи (чипы над результатами):
на складе / только со скидкой / диапазон цены / сортировка. Редактируются в ⚙,
дефолтные: Alle, På lager, Tilbud, Pris ↑, Pris ↓, Under 500 kr.

**Если появится плашка про CORS** — браузер не пустил запрос к API с чужого домена.
Решение: задеплой `cors-worker.js` на Cloudflare Workers (бесплатно, 5 минут):
1. dash.cloudflare.com → Workers & Pages → Create Worker.
2. Вставь код из `cors-worker.js` → Deploy.
3. В ⚙ страницы: «CORS-proxy-mal» = `https://<имя-воркера>.workers.dev/?url={url}`

Прокси пропускает только запросы к power.no.

**Поля товаров** определяются эвристикой (название/цена/старая цена/картинка/сток) —
если что-то не отображается, пришли кусок JSON из DevTools, поправим маппинг.
